# Kuber Logs

Простой CLI-инструмент для просмотра логов `pm2` из подов Kubernetes.

## Предварительные требования

- Установленный [Node.js](https://nodejs.org/)
- Установленный и настроенный `kubectl`, имеющий доступ к вашему Kubernetes кластеру.

## Установка

1. Клонируйте репозиторий или скачайте исходный код.
2. Установите зависимости:
   ```bash
   npm install
   ```

## Настройка

Для работы скрипта необходимо указать пространство имен (namespace) в Kubernetes.

1. Создайте файл `.env` в корневом каталоге проекта.
2. Добавьте в него следующую строку, заменив `<your-namespace>` на ваше реальное пространство имен:
   ```
   NAMESPACE=<your-namespace>
   ```
3. Создайте ссылку на утилиту, чтобы можно было запустить в любом каталоге.
    ```bash
    npm link
    ```
## Использование

Этот инструмент позволяет просматривать логи двумя способами.

### 1. Автоматическое определение сервиса

Скрипт может автоматически определить имя сервиса на основе имени каталога, в котором он запущен.

- Перейдите в каталог, имя которого совпадает с именем вашего сервиса в Kubernetes.
- Запустите скрипт:
  ```bash
  logs
  ```

### 2. Явное указание сервиса

Вы можете явно указать имя сервиса с помощью флага `-s`.

- Запустите скрипт со следующим аргументом, заменив `<service-name>` на имя нужного сервиса:
  ```bash
  logs -s=<service-name>
  ```

После запуска скрипт найдет соответствующий под и выведет логи из `pm2` в ваш терминал.
